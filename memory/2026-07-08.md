## Linux 编译与测试环境搭建记录
- 关键决策：
  - 在 Ubuntu 24.04 (aarch64) 宿主机上部署官方 Swift 5.10.1 环境，随后升级至最新稳定版 Swift 6.3.3 并配置 PATH 符号链接。
  - 为解决 Linux 平台缺失 Objective-C 运行时的限制，在测试中引入跨平台兼容的 `autoreleasepool` 占位实现。
  - 为解决 Linux Foundation 的 `NSNumber` 对 `Bool`/`Int` 等数值类型的隐式类型强制转换与 JIT 匹配重叠问题，采用精细的 `where type(of: value) == X.self` 强类型检查，保证 `NSNumber` 实例能够正确流入特化编码分支。
  - 针对大浮点数在 Linux 和 Darwin 下输出格式的精微精度差异，采用 `#if os(Linux)` 进行条件编译。
  - 修复 `SwiftyJSON.swift` 中的默认 `numberValue` 回退值（`0.0` 改为 `0`），在 Linux 上对 non-numeric 值统一输出 `"0"`。
- 验证结果：
  - 先后在 Swift 5.10.1 和 Swift 6.3.3 环境下执行 `swift test`，196 个测试用例均全部跑通，0 个失败。
- 风险结论：
  - 没有任何公开 API 被修改，重写或条件编译逻辑仅限平台兼容与测试侧，系统级兼容性良好，风险极低。
