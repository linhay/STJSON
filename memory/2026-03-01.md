# 2026-03-01

## STJSON issue #1 处理记录
- 关键决策：`JSONRPC.Response` 在解码时允许缺省 `jsonrpc` 字段，并回填为 `"2.0"`；若显式提供但不是 `"2.0"` 仍按 `invalidRequest` 拒绝。
- 实施方式：仅修改 `Response.init(from:)` 的 `jsonrpc` 读取逻辑（`decodeIfPresent ?? "2.0"`），不新增公开 API，不改请求侧严格校验。
- 验证结果：新增 2 个测试（缺省 `jsonrpc` 兼容解码 + 非法版本拒绝）并通过全量测试（191 tests passed）。
- 风险结论：兼容范围仅限响应解码，可能放宽对极少数非规范响应生产者的容忍度；但仍保留显式版本校验，风险可控。
