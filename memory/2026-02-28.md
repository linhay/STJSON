# 2026-02-28

## STJSON 新增 JSON-RPC 2.0（纯协议层）
- 决策：仅实现 JSON-RPC 2.0 协议层（模型、编解码、校验），不绑定 HTTP/WebSocket。
- 规范策略：严格遵循 JSON-RPC 2.0 核心约束。
  - `jsonrpc` 必须为 `"2.0"`
  - `params` 仅支持 object/array
  - `method` 不能为空且禁止 `rpc.` 前缀
  - `Response` 中 `result` 与 `error` 必须二选一
  - 空 batch 视为 `invalidRequest(-32600)`
  - 通知（无 `id`）不需要响应
- 类型与能力：新增 `JSONRPC.ID / Params / Request / Response / ErrorObject / ErrorCode / Inbound / ProtocolError`。
- coder 约束：JSON-RPC 默认编码/解码入口使用新建 `JSONEncoder()` / `JSONDecoder()`，不依赖可变 `shared`，并增加隔离性测试。
- 测试：新增 `JSONRPCTests` 12 个场景；全量 `swift test` 通过（189 tests）。
- 文档：新增 JSON-RPC BDD 规格与测试矩阵；更新 STJSON skill 与 references 索引并新增 `jsonrpc2.md` 示例。
