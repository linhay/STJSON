## 解决 macOS 15 平台 Double 格式化断言失败记录
- 关键决策：
  - **放宽浮点数序列化断言**：
    - 在 [BaseTests.swift](file:///workspace/open-sources/STJSON/Tests/SwiftJSONTests/BaseTests.swift#L190-L194) 的 `testNumberPrint` 测试中，原先通过 `#if os(Linux)` 来区分 Linux 平台输出 `"-9.999999992e+24"` 与 macOS 平台输出 `"-9.999999991999999e+24"` 的写法十分脆弱。
    - 随着 macOS 15 SDK 的升级与 Swift 6.0/Foundation 底层对高精度 Double 转 String 算法的改进，macOS 15 平台上的输出也变变为更精确且简短的 `"-9.999999992e+24"`，从而导致了原有 macOS 分支的测试断言失败（CI Run 29000064287 / Job 86058297316）。
    - 决策将这块硬编码平台区分分支代码移除，改为定义一个兼容匹配断言，只要 `stringValue` 为 `"-9.999999991999999e+24"` 或 `"-9.999999992e+24"` 均视为测试通过，极大地提升了跨平台、跨 OS 版本的测试稳定性。
- 评测结论：
  - 成功消除了 macOS 15 上的测试障碍，保证了在各平台（新旧 macOS、Linux）以及不同 Swift 编译器版本下的测试一致性与健壮性。
